# Changelog v1.1.1

### Fixes

- Widget submit endpoint now respects per-project screenshot upload size overrides. Previously, the size gate read only the global settings, so projects with a higher custom limit still hit the global 10 MB cap.
- The `@arantic/bugpin-widget` npm package now ships TypeScript declarations. The build emits a bundled `dist/index.d.ts` (via `vite-plugin-dts` with `rollupTypes`) and `package.json` exposes it through both the `types` field and the `exports["."].types` condition. Consumers that import the package from a TypeScript project no longer hit `Cannot find module '@arantic/bugpin-widget' or its corresponding type declarations`.

### UI Improvements

- Project Settings dialog no longer resizes when switching between vertical tabs. The dialog is locked to a single height and the content area scrolls when a tab's content exceeds the available space.

### Security

- Admin Console test tooling now uses `vitest` and `@vitest/coverage-v8` 4.1.8, resolving the critical Vitest vulnerability affecting versions before 4.1.0 (CVE-2026-47429 / GHSA-5xrq-8626-4rwp).

### Maintenance

- Updated the Admin Console Vitest configuration for Vitest 4 and adjusted the shared jsdom observer mocks so typecheck, tests, and production admin builds continue to pass on the patched test runner.
